# 금융관계 휴면화 예측 전환 설계

## 목표

기존 `customer_churn`의 급감·침묵/관측종료 기반 이탈잠재예측을 종료하고, 36개월 전체 관측 법인만을 대상으로 한 **금융관계 휴면화 예측**으로 1번 파이프라인을 교체한다. CRM 우선순위, 세그먼트 기반 추천, SHAP/AI 보고서의 서비스 흐름은 유지한다.

## 범위와 대상

- 분석 코호트: 2023-01~2025-12의 36개월 모두가 관측된 법인(현재 EDA 기준 3,372개).
- 분석 축: 수신, 여신, 요구불 입·출금, 채널 거래, 카드 사용, 외환 거래, 자동이체, 상품관계폭과 각 축의 활성월 비율·추세·변동성.
- 새 y: 특정 단일 거래 축 또는 기존 급감·침묵 규칙으로 미리 고정하지 않는다. `EDA/36개월 특성.ipynb`에서 다축 장기 추세를 비교한 뒤, 지속적·복합적 약화 패턴을 검증해 정의한다.
- 라벨 단위: 법인×기준월 rolling. 기준월 `t`까지의 정보만 feature로 사용하고, 향후 3~6개월의 다축 휴면화 조건을 y로 판단한다. 정확한 과거/미래 창, 약화 축의 수, 기준값은 EDA 후보 비교 후 확정한다.

## 용어와 운영 출력

| 기존 | 전환 후 |
| --- | --- |
| 이탈잠재예측 | 금융관계 휴면화 예측 |
| 급감·침묵 확률 | 휴면화 확률 |
| 이탈 잠재위험 | 금융관계 약화 위험(또는 휴면위험) |
| 잠재이탈고객 | 휴면화 조기관리 대상 |
| 이탈 유형 | 금융관계 약화 유형 |

`휴면화 확률 = P(새 금융관계 휴면화 y=1 | X_t)`로 출력한다. 기존 사슬계수와 관측종료 확률은 새 위험 점수 구성요소에서 제거한다. 고객가치 대리지표와 결합하는 CRM 점수는 새 휴면위험을 사용한다. 이 점수는 회계상 확정 손실이나 실제 해지 확률이 아니다.

## 유지하는 서비스 흐름

```text
36개월 전체 관측 법인 EDA
→ 다축 금융관계 휴면화 y 후보 탐색·검증·확정
→ rolling 휴면화 예측 모델
→ 휴면위험 × 고객가치 대리지표
→ CRM 우선순위
→ 세그먼트별 접촉·상품 추천
→ SHAP/AI 전략 보고서
```

시간 기반 검증, 기준월 이후 정보의 feature 사용 금지, baseline rule과 모델 비교 원칙은 유지한다.

## 교체 경계

1. `AGENTS.md`, `README.md`, 모델 정의 문서는 새 설계를 프로젝트의 기준으로 설명한다.
2. `customer_churn.md`와 기존 `final_churn_*` 구현은 더 이상 운영 기준으로 사용하지 않으며, 재현을 위해 `legacy/` 아래의 역사적 자료로 이동하거나 명시적으로 폐기 상태로 표기한다.
3. 기존 급감·침묵 라벨, 사슬계수, 2단계 fate 모델과 이를 전제한 세그먼트 검증은 새 파이프라인에서 제거한다.
4. 새 구현은 y 확정 뒤 별도의 중립적 이름(예: `financial_dormancy_labels.py`, `financial_dormancy_model.py`)으로 만든다. 기존 코드의 이름만 바꿔 규칙을 재사용하지 않는다.
5. 프런트엔드의 메뉴·KPI·설명 문구는 ‘이탈’에서 ‘금융관계 휴면화/약화’로 바꾸고, 새 모델 산출물이 생기기 전에는 기존 수치가 실데이터 결과인 것처럼 보이지 않게 한다.

## 검증 기준

- 코호트가 모든 법인에서 정확히 36개월인지 확인한다.
- 각 y 후보에 대해 전체 발생률, 업종·지역·등급·전담여부별 발생률, 축별 전조 추세, 대형 법인 영향(중앙값/윈저라이즈드 비교)을 확인한다.
- 후보 y는 특정 단일 지표의 일시 변동이 아니라, 다축·지속성·기준월 이후성 요건을 만족해야 한다.
- 모델은 시간 기반 train/validation/test로 PR-AUC, Top-K recall, decile lift 및 집단별 안정성을 평가한다.
- feature에 미래 기간의 정보가 들어가지 않는 검사를 자동화한다.

## 비범위

- 현 단계에서 휴면화 y의 임계값·필수 축 수·미래 창을 임의로 확정하지 않는다.
- 실제 해지, 실제 손실, 고객별 손익을 추정했다고 주장하지 않는다.
- 기존 y 기반 산출물의 수치와 새 y 기반 결과를 직접 비교 성과로 제시하지 않는다.
