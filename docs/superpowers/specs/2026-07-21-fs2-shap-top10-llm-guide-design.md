# FS2 SHAP Top 10 LLM Guide Design

## Goal

AI 전략 보고서가 현재 운영 모델 `FS2_R1_DACK_DYNAMIC`의 56개 피처와 고객별 Local SHAP Top 10을 정확히 사용하도록 가이드, Gemini 프롬프트 전처리, 생성 결과 후처리를 일치시킨다.

## Fixed Model Contract

- 모델 입력은 R1 관계·규모·구성 피처 16개와 D/A/C/K 동적 피처 40개, 총 56개다.
- 업종, 지역, 고객등급, 전담여부는 모델 피처가 아니며 고객 맥락으로만 사용할 수 있다.
- 고객별 SHAP은 56개 모델 피처의 LightGBM raw-margin 기여도다.
- 보고서 생성 입력은 절대 SHAP이 큰 순서의 Local SHAP Top 10이다.
- UI와 PDF의 SHAP 표에는 Top 10 원본을 순위대로 모두 유지한다.

## SHAP Preprocessing

백엔드는 LLM 호출 전에 `shapFactors`를 다음과 같이 결정론적으로 처리한다.

1. 항목 수가 1~10개인지 확인한다.
2. 순위가 1부터 중복 없이 연속하는지 확인하고 순서대로 정렬한다.
3. SHAP 값이 유한한 숫자인지 확인한다.
4. 실제 56개 피처 이름만 허용한다.
5. 피처를 입출금, 자동이체, 채널, 카드, 수신, 여신, 관계 폭·구성 그룹으로 분류한다.
6. `impact > 0`은 위험 증가 방향, `impact < 0`은 보호 방향, `impact == 0`은 중립으로 표시한다.
7. Top 10 절대 SHAP 합계를 분모로 각 항목과 그룹의 `top10_abs_share`를 계산한다. 이 값은 56개 전체 기여율이나 위험확률 변화량으로 표현하지 않는다.
8. 동일 그룹의 피처는 signed 합, absolute 합, 대표 피처, 포함 순위를 함께 제공해 LLM이 하나의 신호로 합쳐 서술할 수 있게 한다.

프롬프트에는 원본 Top 10과 그룹 요약을 모두 전달한다. 원본은 근거 보존과 순위 확인용이고, 그룹 요약은 반복 문장 방지용이다.

## Feature Translation

동적 피처의 실제 한국어 접미사를 기준으로 비교기간과 의미를 번역한다.

- `최근3대직전3_로그차이`: 최근 3개월과 직전 3개월 비교
- `최근3대이전9_로그차이`: 최근 3개월과 이전 9개월 비교
- `H2대H1_로그차이`: 최근 반기와 앞선 반기 비교
- `로그변동성`: 최근 12개월 활동 변동성
- `TheilSen_추세`: 최근 12개월 강건 추세
- `최근3대이전9_활성률차이`: 활동월 빈도 변화
- `월간50퍼감소횟수`: 최근 12개월 월간 50% 이상 감소 횟수
- `월간감소율`: 전월보다 감소한 월의 비율
- `최대연속비활성개월`: 최장 연속 비활성 기간
- `현재월대직전6_로그차이`: 현재월과 직전 6개월 평균 비교

피처값이 없는 경우 LLM은 SHAP 부호와 번역된 피처 의미만 사용하며 실제 금액, 비율, 증감 방향을 추정하지 않는다.

## Prompt Rules

Gemini는 다음 고정 규칙을 따른다.

- 검증된 컨텍스트와 전처리된 SHAP 근거만 사용한다.
- Local SHAP Top 10 원본을 모두 입력으로 받는다.
- 같은 그룹의 유사 피처는 보고서 문장에서 하나의 종합 신호로 묶는다.
- 위험요인은 양수 SHAP, 보호요인은 음수 SHAP에 근거한다.
- raw SHAP을 확률 변화량이나 인과효과로 표현하지 않는다.
- 업종, 지역, 고객등급, 전담여부를 모델 판단의 직접 원인으로 표현하지 않는다.
- 실제 해지, 부도, 확정 휴면 또는 확정 손실로 단정하지 않는다.
- `CLV_Risk`와 `PotentialLoss`는 시나리오 지표로만 설명한다.
- 추천은 RM 확인 후 검토할 초기 접촉 전략으로 표현한다.

응답 JSON 스키마는 기존 `GeminiNarrative`를 유지해 프론트엔드와 PDF 계약을 변경하지 않는다.

## Narrative Postprocessing

LLM 응답을 스키마 검증한 뒤 백엔드가 다음 안전 규칙을 적용한다.

- 실제 해지·부도·확정 휴면 확률로 오해시키는 고정 표현을 지속거래약화 가능성 표현으로 정규화한다.
- raw SHAP 수치를 확률 변화로 연결하는 문장은 허용하지 않는다.
- 필수 한계문구 두 가지를 `caveats`에 보장한다.
  - Y는 실제 해지·부도·확정 휴면이 아닌 지속거래약화 proxy 예측이다.
  - SHAP은 인과관계나 확률 변화량이 아닌 모델 예측 기여도다.
- `CLV_Risk`와 `PotentialLoss`가 확정 손실이 아니라는 문구를 보장한다.
- 중복 한계문구는 정규화 과정에서 제거한다.

후처리는 새로운 정량값이나 거래사실을 만들지 않는다. 안전하게 고칠 수 없는 금지 표현이나 SHAP 방향 위반이 발견되면 보고서 생성을 실패시켜 기존 502 오류 계약을 유지한다.

## Data Flow

```text
서비스 DB의 검증된 고객 컨텍스트와 SHAP Top 10
→ 56개 피처 계약 검증
→ 피처 번역·방향 판정·그룹 요약
→ 원본 Top 10 + 그룹 요약을 Gemini에 전달
→ GeminiNarrative 스키마 검증
→ 금지 표현 검사·필수 한계문구 보장
→ 기존 GeneratedReport 조립
→ UI/PDF에 원본 SHAP Top 10 전체 표시
```

## Error Handling

- 알 수 없는 피처, 비연속 순위, 비유한 SHAP 값은 LLM 호출 전에 `ReportGenerationError`로 실패한다.
- Gemini의 비정상 JSON 또는 스키마 불일치는 기존과 같이 `ReportGenerationError`로 감싼다.
- 안전 후처리에서 확실히 교정할 수 없는 위험한 표현은 조용히 노출하지 않고 생성 실패로 처리한다.
- 외부 응답과 예외의 민감한 상세는 API 오류 메시지에 노출하지 않는다.

## Testing

- 56개 피처가 모두 허용되고 이전 `CAT_*` 피처가 거부되는지 검증한다.
- Top 10 원본이 모두 프롬프트에 남고 그룹 요약이 생성되는지 검증한다.
- 같은 D/A/C/K 축의 여러 피처가 하나의 그룹으로 집계되는지 검증한다.
- 양수·음수·0 SHAP 방향과 Top 10 내부 상대기여도가 정확한지 검증한다.
- 필수 proxy·SHAP·시나리오 한계문구가 후처리에서 보장되는지 검증한다.
- 안전하게 교정할 수 없는 SHAP 확률 표현과 방향 위반이 거부되는지 검증한다.
- 기존 API 응답, UI, PDF가 SHAP 10개를 계속 반환·표시하는 회귀 테스트를 유지한다.

## Non-goals

- LightGBM 모델 또는 SHAP 값을 재계산하지 않는다.
- SHAP Top 10 선정 로직을 변경하지 않는다.
- 프론트엔드와 PDF의 보고서 응답 스키마를 변경하지 않는다.
- LLM이 원시 월별 데이터로 변화율이나 금액을 새로 계산하게 하지 않는다.
