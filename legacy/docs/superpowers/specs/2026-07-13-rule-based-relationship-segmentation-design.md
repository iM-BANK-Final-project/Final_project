# Rule-Based Relationship Segmentation Design

## 1. Purpose

`segmentation_final_report.md`에서 잠근 `L30_H70_M15` 법인 관계 세그먼트를 재현 가능한 Python 모듈과 실행 명령으로 구현한다.

세그먼트는 거래약화 위험등급이나 고객가치 등급이 아니다. 거래활동·수신·여신 중 법인이 어떤 관계구조를 가지는지 설명하고, 하나의 전역 지속거래약화 예측모델과 RM 대응전략에 고객 맥락을 제공한다.

## 2. Source And Time Contract

- 원천 입력은 월별 법인 CSV다.
- 분석 대상은 `2023-01~2025-12`가 중복 없이 모두 존재하는 36개월 완전관측 법인이다.
- 세그먼트 기준분포와 룰은 `2023-01~2023-12` 정보만으로 만든다.
- `2024-01~2024-12`는 잠긴 2023 기준을 적용한 사후 안정성 검증에만 사용한다.
- 2025년 정보와 `Y_지속거래약화_3M70`은 세그먼트 생성·임계값·라벨 결정에 사용하지 않는다.
- 결측 금액을 0으로 대체하지 않는다.
- 음수 금액, 법인×월 중복, 불완전한 기준월은 오류로 중단한다.

## 3. Relationship Axes

월별 세 관계축은 다음 원천 금액의 합으로 계산한다.

```text
거래활동금액
= 요구불입금금액 + 요구불출금금액
 + 창구거래금액 + 인터넷뱅킹거래금액
 + 스마트뱅킹거래금액 + 폰뱅킹거래금액 + ATM거래금액
 + 신용카드사용금액 + 체크카드사용금액

수신관계금액
= 요구불예금잔액 + 거치식예금잔액 + 적립식예금잔액
 + 수익증권잔액 + 신탁잔액 + 퇴직연금잔액

여신관계금액
= 여신_운전자금대출잔액 + 여신_시설자금대출잔액
```

각 월별 합은 구성 원천이 모두 관측된 경우에만 계산한다. 축마다 `log1p`를 적용하고 법인별 12개월 중앙값을 `거래활동관계수준`, `수신관계수준`, `여신관계수준`으로 사용한다. 평균, 원 금액, 추세, Y는 주 세그먼트 점수에 넣지 않는다.

## 4. Frozen Percentile Mapping

2023년 법인별 세 관계수준으로 축별 기준분포를 만든다. 기준연도 자체의 percentile은 평균순위 방식으로 계산한다.

```text
percentile = pandas.rank(method="average", pct=True)
```

동일한 값은 같은 평균 percentile을 갖는다. 이후 기간은 새 기간 안에서 다시 순위를 매기지 않고 2023 기준분포의 경험적 percentile에 매핑한다.

- 이후 기간의 값은 동률을 포함해 그 값 이하인 2023 관측치 비율(`searchsorted(..., side="right") / n`)을 사용한다.
- 이 right-continuous 경험적 CDF가 보고서의 2024 유지율과 ARI를 재현한다.
- 기준 최솟값보다 작으면 `0.0`, 최댓값보다 크면 `1.0`으로 제한한다.

2023 기준분포의 세 축 원점수를 CSV로 저장해 이후 scoring에 재사용할 수 있게 한다.

## 5. L30_H70_M15 Assignment

세 축 percentile을 `거래활동점수`, `수신관계점수`, `여신관계점수`로 정의한다. 룰 적용 우선순위는 다음과 같다.

1. 세 점수가 모두 `0.30 이하`면 `저관계`다.
2. 세 점수 중 두 개 이상이 `0.70 이상`이면 `복합고관계`다.
3. 최고 점수가 차순위 점수보다 `0.15 이상` 높으면 최고 축에 따라 `거래활동중심`, `수신중심`, `여신중심`을 부여한다.
4. 나머지는 `균형·중간관계`다.

경계값 `0.30`, `0.70`, `0.15`는 각각 조건에 포함한다. 복합고관계와 단일축 중심 조건이 동시에 가능한 경우 복합고관계를 우선한다. 동률 최고점은 dominance margin을 충족할 수 없으므로 균형·중간관계가 된다.

## 6. Components

### 6.1 Core Module

`src/segmentation/relationship_segments.py`가 다음 책임을 가진다.

- 설정과 원천 컬럼 계약 정의
- 완전관측 코호트 및 금액 검증
- 월별 세 관계축 생성
- 임의의 12개월 구간에 대한 법인별 관계수준 요약
- 2023 기준 percentile fit
- 고정 기준분포를 이용한 이후 구간 score
- `L30_H70_M15` 세그먼트 할당
- 세그먼트 구성비와 연도 간 안정성 요약

각 단계는 독립 함수로 두어 계산과 파일 입출력을 분리한다.

### 6.2 Runner

`src/segmentation/run_relationship_segments.py`는 원천 CSV를 한 번 읽고 다음 파일을 저장한다.

```text
relationship_segment_reference_2023.csv
relationship_segment_assignments_2023.csv
relationship_segment_profile_2023.csv
relationship_segment_assignments_2024.csv
relationship_segment_profile_2024.csv
relationship_segment_stability_2023_2024.csv
```

assignment에는 법인ID, 세 축 관계수준, 세 축 percentile 점수, 최종 세그먼트를 포함한다. stability에는 전체 동일 세그먼트 유지율, adjusted Rand index, 세그먼트별 유지율, 구성비 변화를 포함한다.

## 7. Validation And Acceptance

합성 데이터 단위 테스트에서 다음을 검증한다.

- 세 축 원천 합과 `log1p` 12개월 중앙값
- 결측을 0으로 대체하지 않는 동작
- 음수·중복·불완전 월 오류
- 평균순위 percentile과 동률 처리
- 2024 scoring에서 2023 기준분포가 고정되는 동작
- 네 단계 룰 우선순위와 정확한 경계값
- 여섯 세그먼트 label 계약
- runner 산출물 계약

실데이터 실행에서는 다음 보고서 수치를 재현 기준으로 확인한다.

| 세그먼트 | 기대 법인 수 |
|---|---:|
| 복합고관계 | 914 |
| 균형·중간관계 | 835 |
| 여신중심 | 717 |
| 거래활동중심 | 332 |
| 수신중심 | 315 |
| 저관계 | 259 |

2024 사후 검증은 보고서의 전체 유지율 `80.01%`, ARI `0.5866`, 최대 구성비 변화 `1.275%p`와 대조한다. 구현의 명시적 percentile 동률 처리 차이로 수치가 다르면 코드를 결과에 맞춰 임의 변경하지 않고 차이의 원인을 먼저 감사한다.

## 8. Scope Boundaries

- KMeans와 DBSCAN을 다시 실행하지 않는다.
- 업종·지역·고객등급·전담여부를 세그먼트 입력으로 사용하지 않는다.
- 세그먼트별 별도 예측모델을 만들지 않는다.
- Y 비율이나 모델 성능으로 세그먼트 임계값을 조정하지 않는다.
- 고변동·수신다상품·여신다상품·다채널·카드활성·외환활성 modifier는 보고서에 정확한 산식과 임계값이 모두 명시되어 있지 않으므로 핵심 세그먼트 v1 라벨에 포함하지 않는다.
- 운영 월별 rolling 갱신은 동일한 12개월 요약·고정 percentile scoring API를 재사용하되, 이번 runner 산출 범위는 보고서의 2023 fit과 2024 사후 검증까지로 제한한다.

## 9. Downstream Contract

예측모델에는 하나의 전역 모델을 유지하면서 다음만 선택적으로 결합한다.

```text
관계세그먼트
+ 거래활동점수
+ 수신관계점수
+ 여신관계점수
```

세그먼트 라벨은 위험확률을 대체하지 않으며, SHAP 설명과 RM 대응전략에서 고객 관계맥락으로 사용한다.
