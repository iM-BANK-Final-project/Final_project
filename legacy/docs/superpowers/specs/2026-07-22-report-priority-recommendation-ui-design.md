# 보고서·CRM 우선순위·맞춤추천 UI 개선 설계

## 목표

- AI 리포트 페이지 제목에서 `저장된` 표현을 제거한다.
- CRM 우선순위 표의 방어순위 값을 첫 번째 열 안에서 가운데 정렬한다.
- 맞춤추천 페이지에 기존 지속거래약화 예측 페이지와 같은 위험도 선택 필터를 추가한다.

## 화면 변경

- AI 리포트 제목은 `지속거래약화 전략 보고서`로 표시한다. 저장 여부를 암시하는 다른 동작은 추가하지 않는다.
- CRM 우선순위 표는 첫 번째 헤더와 데이터 셀만 가운데 정렬한다. 다른 열의 정렬과 너비는 유지한다.
- 맞춤추천 필터 영역에 `위험도 전체`를 기본값으로 하는 선택 상자를 추가한다. 선택지는 공통 `/api/filter-options` 응답의 `riskLevels`를 사용한다.

## API와 데이터 흐름

- 프론트엔드는 선택한 위험도를 `/api/recommendations`의 `risk_level` 쿼리 파라미터로 전달한다.
- 백엔드 라우트와 저장소 조회 함수에 nullable `risk_level`을 추가한다.
- 저장소는 기존 공통 필터 생성기를 통해 `customer_snapshots.risk_level`을 조건에 포함한다.
- 빈 값은 필터를 적용하지 않으며 기존 세그먼트·약화유형 필터와 함께 AND 조건으로 동작한다.

## 오류와 호환성

- 기존 API 호출은 `risk_level`을 생략할 수 있으므로 호환성을 유지한다.
- 필터 옵션 또는 추천 조회 실패 시 현재 로딩·오류·재시도 UI를 그대로 사용한다.

## 검증

- AI 리포트 제목의 새 문구를 화면 테스트로 검증한다.
- CRM 첫 번째 열의 가운데 정렬 규칙을 스타일 테스트로 검증한다.
- 맞춤추천 위험도 선택 시 API가 `risk_level`을 전달하는지 프론트 테스트로 검증한다.
- 백엔드 API·저장소 테스트에서 위험도 조건이 실제 결과를 제한하는지 검증한다.
- 전체 프론트·백엔드 테스트와 프론트 프로덕션 빌드를 실행한다.
